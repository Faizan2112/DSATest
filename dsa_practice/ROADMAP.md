# The Ultimate DSA Roadmap for Interview Prep

This roadmap is designed to take you from a beginner level to clearing technical interviews at top tech companies.

## üèÜ Success Mantra
1.  **Consistency**: Code every day (even if it's just 1 problem).
2.  **Pattern Recognition**: Don't memorize code; understand the underlying patterns (e.g., Sliding Window, Two Pointers).
3.  **Spaced Repetition**: Revisit problems you found hard after 3-4 days.

---

## üìÖ Phase 1: The Foundations (Weeks 1-2)
**Goal**: Get comfortable with language syntax and basic complexities.

### Topics
*   **Language Mastery**: Java, C++, or Python. (Stick to one).
*   **Time & Space Complexity**: Big O notation, Best/Average/Worst case.
*   **Basic Math**: GCD, LCM, Primes, Pallindromes.

### Practice Questions
1.  **Fizz Buzz** (Easy)
2.  **Count Primes** (Easy)
3.  **Power of Three** (Easy)
4.  **Reverse Integer** (Medium)

---

## üß± Phase 2: Linear Data Structures (Weeks 3-5)
**Goal**: Master storing and manipulating linear data.

### 1. Arrays & Strings
*   **Concepts**: Dynamic arrays, resizing, String mutability.
*   **Questions**:
    *   [Easy] Two Sum
    *   [Easy] Valid Anagram
    *   [Easy] Contains Duplicate
    *   [Medium] Product of Array Except Self
    *   [Medium] Maximum Subarray (Kadane‚Äôs Algo)
    *   [Medium] Group Anagrams
    *   [Medium] Longest Consecutive Sequence

### 2. Two Pointers & Sliding Window
*   **Concepts**: Shrink/Expand windows, left/right pointers.
*   **Questions**:
    *   [Easy] Valid Palindrome
    *   [Medium] 3Sum
    *   [Medium] Container With Most Water
    *   [Medium] Best Time to Buy and Sell Stock
    *   [Medium] Longest Substring Without Repeating Characters

## üîó Phase 3: Linked Lists, Stacks, & Queues (Weeks 6-8)

### 1. Linked Lists
*   **Concepts**: Reversing, Cycle Detection (Floyd‚Äôs), Merging.
*   **Questions**:
    *   [Easy] Reverse Linked List
    *   [Easy] Merge Two Sorted Lists
    *   [Easy] Linked List Cycle
    *   [Medium] Reorder List
    *   [Medium] Remove Nth Node From End of List

### 2. Stacks & Queues
*   **Concepts**: LIFO/FIFO, Monotonic Stack.
*   **Questions**:
    *   [Easy] Valid Parentheses

## üéã Phase 4: Non-Linear Data Structures (Weeks 9-11)
**Goal**: Hierarchical data and searching.

### 1. Trees & BST
*   **Concepts**: Recursion, BFS (Level Order), DFS (In/Pre/Post-order).
*   **Questions**:
    *   [Easy] Invert Binary Tree
    *   [Easy] Maximum Depth of Binary Tree
    *   [Easy] Same Tree
    *   [Easy] Subtree of Another Tree
    *   [Medium] Binary Tree Level Order Traversal
    *   [Medium] Validate Binary Search Tree
    *   [Medium] Kth Smallest Element in a BST
    *   [Medium] Construct Binary Tree from Preorder and Inorder Traversal

### 2. Heaps (Priority Queue)
*   **Concepts**: Min-Heap/Max-Heap, Top K elements.
*   **Questions**:
    *   [Easy] Kth Largest Element in a Stream
    *   [Medium] K Closest Points to Origin
    *   [Medium] Task Scheduler
    *   [Hard] Find Median from Data Stream

## üß† Phase 6: Advanced Algorithms (Weeks 15-18)

### 1. Dynamic Programming (DP)
*   **Concepts**: Memoization (Top-Down) vs Tabulation (Bottom-Up). 1D and 2D DP.
*   **Questions**:
    *   [Easy] Climbing Stairs
    *   [Medium] House Robber
    *   [Medium] House Robber II
    *   [Medium] Longest Palindromic Substring

    *   [Medium] Coin Change
    *   [Medium] Longest Increasing Subsequence
    *   [Medium] Unique Paths (2D)
    *   [Medium] Longest Common Subsequence

### 2. Backtracking
*   **Concepts**: Exploring all paths, Pruning.
*   **Questions**:
    *   [Medium] Subsets
    *   [Medium] Combination Sum
    *   [Medium] Permutations
    *   [Medium] Word Search

### 3. Greedy & Intervals
*   **Concepts**: Local optimum key. Intervals merging.
*   **Questions**:
    *   [Medium] Jump Game
    *   [Medium] Insert Interval
    *   [Medium] Merge Intervals
    *   [Medium] Non-overlapping Intervals
